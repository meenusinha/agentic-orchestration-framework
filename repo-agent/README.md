# Repo Agent — Quick Reference

Copy the contents of this directory into your repo, then edit `mcp/config.yaml`.
After that, run `orchestrator/setup.py` — it generates `.vscode/mcp.json` and
`.github/copilot-instructions.md` for you automatically.

See `framework/README.md` for full step-by-step instructions.

---

## Files you copy into your repo

```
mcp/
├── config.yaml        ← EDIT THIS (repo_name, components, src_path)
├── mcp_server.py      ← do not edit
├── repo_rag.py        ← do not edit
└── demo_logger.py     ← do not edit
knowledge/
└── README.md          ← DROP YOUR .md DOCS HERE
.github/
└── copilot-instructions.md   ← generated by setup.py; do not hand-edit
```

---

## config.yaml fields

| Field | Required | Description |
|-------|----------|-------------|
| `repo_name` | Yes | Unique id — must match orchestrator/mcp/config.yaml |
| `display_name` | Yes | Human-readable name shown in routing output |
| `components` | Yes | Key classes / modules (shown to the AI) |
| `src_path` | No | Path to source code to index. Set `null` to skip. |
| `knowledge_path` | No | Where to find .md docs. Default: `./knowledge` |
| `extra_extensions` | No | Additional file extensions to index, e.g. `[".sol"]` |

---

## Supported source file extensions (indexed by default)

`.py` `.js` `.ts` `.jsx` `.tsx` `.java` `.go` `.rs` `.cpp` `.c` `.h` `.cc` `.cxx` `.cs` `.rb` `.swift` `.kt` `.php` `.scala` `.thrift` `.proto` `.graphql`

## Re-indexing

Delete `.chroma_db/` to force a full rebuild on next MCP server start:

```bash
rm -rf .chroma_db/
```
